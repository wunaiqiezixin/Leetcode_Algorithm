# 739. 每日温度
[力扣题目链接](https://leetcode.cn/problems/daily-temperatures/description/)

---
>单调栈
## 思路
维护一个单调栈  
也就是小的元素是靠近栈顶，栈中的元素有序  
不太一样的是，本题栈中应该存放的是元素的下标，因为要算当前元素与下一个元素的下标差，知道了下标就知道了下标对应的值，没有区别  
我们从后往前遍历数组，这很关键  
每遍历到一个元素，我们就要在栈中找比它大的下一个元素的下标，因为我们已经遍历过它后面的所有元素了，或者说栈中已经存放着所有可能比当前元素大的元素的下标  
当前元素与栈顶元素比较，如果栈顶下标对应的元素小于或等于当前元素，出栈，因为这时候栈顶对应的下标已经没有意义了(它的前面已经有了比它大的元素)  
如果找得到，即栈最后不为空，记录  
如果没有找到，记录为0  
很关键的是，不管找没找到，都要把当前元素的下标入栈，因为我们不能确定当前元素与前面的元素的大小关系，也就是说当前元素可能会大于前边的元素  

### ***C++***
```CPP
class Solution {
public:
    vector<int> dailyTemperatures(vector<int>& temperatures) {
        int n = temperatures.size();
        vector<int> answer(n, 0);/* 初始化为0 */
        stack<int> stk; /* 小的元素靠近栈顶, 记录元素下标 */
        /* 从后往前遍历 */
        int i;
        for (i = n - 1; i >= 0; --i)
        {
            /* 找比当前元素大的下一个元素的下标 */
            while (!stk.empty() && 
            temperatures[i] >= temperatures[stk.top()]) /* 注意是 >= */
                stk.pop();
            if (!stk.empty())/* 找到了，记录它 */
                answer[i] = stk.top() - i;
            stk.push(i); /* 找没找到都将当前元素入栈，因为有可能比前面的元素大 */
        }
        return answer;
    }
};
```
